Basic Input Property

Passing data from parent to child

Child Component (user.component.ts)

  import {{ '{' }} input {{ '}' }} from '@angular/core';
  
  export class UserComponent {{ '{' }}
    // Define input property
    name = input<string>();
    
    // Input with default value
    occupation = input('Developer');
    
    // Use in template
    // {{ '{{ name() }}' }} {{ '{{ occupation() }}' }}
  {{ '}' }}

Parent Component Template (app.component.html)

  <!-- Pass values to child -->
  <app-user 
    [name]="currentUserName"
    [occupation]="currentUserOccupation">
  </app-user>
  
  <!-- Direct string value -->
  <app-user name="Simran"></app-user>

Advanced Input Features

Required inputs, transforms, and complex types

Required Inputs

  // Child component
  userId = input.required<number>();
  
  // Parent MUST provide this
  <app-user [userId]="123"></app-user>
  
  // TypeScript will enforce this
  // Missing userId causes error

‚úÖ Required inputs ensure parent components provide essential data.

‚ùå Missing required inputs cause TypeScript compilation errors.

Input Transforms

  // Transform input values
  salary = input(0, {{ '{' }}
    transform: (value: string | number) => {{ '{' }}
      // Convert string to number
      const num = typeof value === 'string' 
        ? parseFloat(value) 
        : value;
      return Math.round(num);
    {{ '}' }}
  {{ '}' }});
  
  // Parent can pass string or number
  <app-user salary="85000"></app-user>
  <app-user [salary]="85000"></app-user>

üîÑ Transforms allow data conversion/validation on input.

üéØ Useful for formatting, validation, or type conversion.

Complex Object Types

  // Complex object type
  address = input<{{ '{' }}city: string, country: string{{ '}' }}>({{ '{' }}
    city: 'Unknown',
    country: 'Unknown'
  {{ '}' }});
  
  // Pass object from parent
  currentUserAddress = {{ '{' }}
    city: 'San Francisco',
    country: 'USA'
  {{ '}' }};
  
  <app-user [address]="currentUserAddress"></app-user>

üè¢ Objects allow grouping related data together.

üì¶ Pass complex data structures as single input.

Array Inputs

  // Array type input
  skills = input<string[]>([]);
  
  // Pass array from parent
  currentSkills = ['Angular', 'TypeScript', 'RxJS'];
  
  <app-user [skills]="currentSkills"></app-user>
  
  // Use in child template
  @ (skill of skills(); track $index) {{ '{' }}
    <span>{{ '{{ skill }}' }}</span>
  {{ '}' }}

üìã Arrays allow passing lists of data.

üé® Perfect for skills, tags, items, or any list data.

üìö Input Syntax Reference

Basic Input

  // Child component
  name = input<string>();
  
  // Parent template
  <app-child [name]="value"></app-child>
  
  // Child template
  {{ '{{ name() }}' }}

Default Value

  // With default value
  title = input('Untitled');
  
  // Parent can override or omit
  <app-child></app-child>
  // Uses "Untitled"
  
  <app-child [title]="'My Title'"></app-child>
  // Uses "My Title"

Required Input

  // Required input
  id = input.required<number>();
  
  // Parent MUST provide
  <app-child [id]="123"></app-child>
  
  // TypeScript error if missing

Input Transform

  // With transform
  count = input(0, {{ '{' }}
    transform: (value: string | number) => {{ '{' }}
      return Number(value);
    {{ '}' }}
  {{ '}' }});
  
  // Parent can pass string
  <app-child count="42"></app-child>
  // Child receives number 42

üÜö Old vs New Syntax

Old Syntax (Decorator)

  // Angular 16 and earlier
  import {{ '{' }} Component, Input {{ '}' }} from '@angular/core';
  
  @Component({{ '{' }}...{{ '}' }})
  export class UserComponent {{ '{' }}
  @Input() name?: string;
  @Input() occupation = 'Developer';
    
    // Use directly in template
    // {{ '{{ name }}' }}
  {{ '}' }}

New Syntax (Signal-based)

// Angular 17+
import {{ '{' }} Component, input {{ '}' }} 
from '@angular/core';
  
@Component({{ '{' }}...{{ '}' }})
  export class UserComponent {{ '{' }}
    name = input<string>();
    occupation = input('Developer');
    
    // Call as function in template
    // {{ '{{ name() }}' }}
  {{ '}' }}

‚úÖ Advantages of New Syntax:

  • ‚Ä¢ Better TypeScript support
  • ‚Ä¢ Required inputs with input.required()
  • ‚Ä¢ Built-in transforms
  • ‚Ä¢ Signals integration
  • ‚Ä¢ No decorators needed

üìö Key Takeaways

üì•

Input Definition

Use input() function to define properties that receive data from parent components.

üîó

Property Binding

Parent passes data using [property]="value" syntax in template.

‚ö°

Signal-based

Inputs return signals. Access values with property() function call.

üéØ

Reusability

Same component can be used multiple times with different input data.
